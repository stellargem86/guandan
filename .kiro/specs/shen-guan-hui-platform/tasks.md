# Implementation Plan: shen-guan-hui-platform

## Overview

本实现计划将深掼会平台从技术设计转化为可执行的编码任务。平台包含三个子项目：uni-app 移动端（微信小程序）、Vue 3 Web 管理后台、Python FastAPI 后端服务。实现分为 16 个阶段，从基础架构搭建开始，逐步构建认证、支付、LBS、内容、赛事、ELO 排行等核心模块，最终完成管理后台和测试部署。

## Tasks

- [ ] 1.1 初始化 uni-app 移动端项目 (Vue 3 + Vite + Tailwind CSS + TypeScript)

- [ ] 1.3 初始化 FastAPI 后端项目 (项目结构、依赖管理、配置系统)
- [ ] 1.4 配置 PostgreSQL 数据库连接和 Alembic 迁移框架

## Notes

- 技术栈：前端 uni-app (Vue 3 + Vite + Tailwind CSS)，后端 Python FastAPI，数据库 PostgreSQL + Redis
- 所有支付场景均通过微信支付 JSAPI 实现，分账使用微信支付分账 API
- LBS 地理位置使用 Redis GEO 命令实现高频近距离查询（<100ms）
- ELO 算法使用标准 ELO 公式，K-factor 新手32/成熟玩家16，段位阈值可配置
- 前端移动端优先微信小程序，通过 uni-app 保持跨平台兼容性
- 管理后台采用 SaaS 多角色门户设计：超级管理员、商户、组织者三级入口
- 所有 API 采用 JWT 认证 + RBAC 权限控制，支持 Token 黑名单机制
- 内容审核采用先发后审 + 敏感词过滤 + 人工复审三级机制
- Tasks marked with `*` are optional and can be skipped for faster MVP
- Checkpoints ensure incremental validation
